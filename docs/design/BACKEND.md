## 一、技术栈定稿

### 1. 数据库：Neon（Serverless PostgreSQL）

**是什么**：一个"放在云上"的 PostgreSQL 数据库。PostgreSQL 是全球最流行的关系型数据库之一，专门为电商这种"订单/库存要绝对准确"的场景设计。

**为什么选 Neon（而非 Supabase / 自建）**：
- ✅ **scale-to-zero**：没人访问时不收费，初期月费 $0
- ✅ **数据库分支**：我能给每个功能开发单独建一个数据库副本，改坏了不影响线上（对你无感，但对我开发很关键）
- ✅ **Vercel 原生集成**：一键连接，零配置
- ✅ **强事务**：订单/库存/优惠券需要"绝对准确"，PostgreSQL 的强一致性是电商刚需
- ✅ **免费额度**：0.5GB 存储 + 每月足够小时数，够用到几百单/月

**备选（不选的原因）**：
- Supabase：功能更全（含 Auth/Storage），但和 Payload/Better Auth 功能重叠，反而复杂
- Turso (SQLite)：免费额度大，但 SQLite 处理"并发下单扣库存"不如 PostgreSQL 稳妥
- 自建数据库：要运维，你不懂代码 + 我不在场时出问题没人管，否决

**成本**：免费起步；流量大了约 $19/月起。

**需要你做**：注册一个 Neon 账号（免费），把数据库连接字符串给我。我会指导你注册。

---

### 2. ORM：Drizzle

**是什么**：我写 TypeScript 代码操作数据库的工具。"ORM"= 把数据库的表翻译成代码里的对象，让我用类型安全的方式查数据。

**为什么选 Drizzle（而非 Prisma）**：
- ✅ **TypeScript 原生**：和我们的 Next.js 代码无缝融合
- ✅ **轻量**：bundle 小，在 Vercel serverless 上启动快（直接影响页面打开速度）
- ✅ **无 codegen 步骤**：改数据库结构后不用跑额外命令，开发更快
- ✅ **性能基准优于 Prisma 约 10 倍**（社区基准测试）

**备选**：Prisma 7（更成熟、文档更好，但重一些）。如果 Drizzle 踩坑严重，可随时切换。

**成本**：免费（开源）。

**需要你做**：无。纯开发工具。

---

### 3. 后台 CMS：Payload CMS 3（Postgres adapter）

**是什么**：你登录后看到的管理后台——录商品、传图、看订单、改库存、发优惠券，全在这个界面完成。**这是你日常用的最多的工具。**

**为什么选 Payload（而非 Shopify Admin / 自研 / Sanity）**：
- ✅ **原生嵌进 Next.js**：后台和你看到的网站是同一个应用，不用维护两套系统
- ✅ **自带 Admin UI**：开箱即用的管理界面（表格、表单、图片上传），我配好字段你直接用
- ✅ **Local API**：网站前台直接读数据库拿商品数据，零网络延迟，SEO 和速度极佳
- ✅ **TS schema-as-code**：字段定义即类型，我改字段你立刻在后台看到
- ✅ **直连 Neon**：用 Postgres adapter，和数据库共用一个实例，不另起库
- ✅ **内置 auth**：后台登录直接用 Payload 的，不用再接认证

**你决定**：先用 Payload 默认面板（界面够用但不华丽），业务跑顺后我再定制美化。

**备选（不选的原因）**：
- 自研 admin：重复造轮子，开发周期翻倍
- Sanity：偏内容运营（博客/营销页），管商品/订单/库存不如 Payload 强

**成本**：免费（开源自托管）。

**需要你做**：我搭好后，给你一个后台网址 + 账号密码，你登录就能用。我会写一份《后台使用手册》教你录商品、看订单。

### 4. 会员认证：Better Auth

**是什么**：处理 C 端顾客的注册、登录、忘记密码、第三方登录（Google/Apple）。

**为什么选 Better Auth（而非 Clerk / Auth.js）**：
- ✅ **自托管、无 MAU 成本**：不按用户数收费（Clerk 按 MAU 收费，用户多了贵）
- ✅ **TypeScript 强类型**：和代码无缝融合
- ✅ **可自定义**：会员积分、订单关联、收藏夹等业务逻辑都能加
- ✅ **邮箱+社交登录都支持**：Google/Apple 一键登录

**备选**：
- Clerk：想最快上线、不在意按用户付费可选。但我建议自托管省钱。

**成本**：免费（开源）。

**需要你做**：无。我接入后，注册/登录页就能用了（页面我已经做好了）。

---

### 5. 支付：Stripe（Embedded Checkout + PayPal）

**是什么**：收钱的。顾客在结账页输入卡号/PayPal，Stripe 处理支付，钱进你的香港公司账户。

**为什么选 Stripe**：
- ✅ **香港公司可直连**：你的香港主体可以直接用 Stripe 港区收款
- ✅ **Embedded Checkout**：支付表单嵌在你的结账页里（不跳转），保品牌体验 + 低 PCI 合规负担
- ✅ **一个集成覆盖多种支付**：信用卡 / Apple Pay / Google Pay / PayPal / Klarna（欧洲分期）
- ✅ **欧美消费者最信任**：Stripe 品牌背书提升转化率
- ✅ **Webhook 同步订单**：支付成功自动更新订单状态，不用手动对账

**关键工程纪律（我会执行）**：
1. **Webhook 签名验证必做**：防止伪造支付通知
2. **幂等处理**：同一笔支付重复通知不重复发货
3. **不依赖前端回调**：以 Stripe webhook 为准，不信任浏览器返回

**货币**：USD 结账（欧美主市场），结算到你的港币/多币种账户。

**成本**：每笔交易 **2.9% + $0.30**（标准国际费率）。无月费。

**需要你做**：
1. 用香港公司资料注册 Stripe 账号（我指导）
2. 准备：公司注册证 (CI)、商业登记证 (BR)、银行账户
3. 把 Stripe 的 API 密钥给我（我接入）

---

### 6. 邮件：Resend + React Email

**是什么**：发交易邮件——订单确认、发货通知、弃单挽回、欢迎邮件。

**为什么选 Resend**：
- ✅ **开发者体验最佳**：和 Next.js 集成最顺
- ✅ **React Email 模板**：邮件模板用 React 写，可复用、可预览
- ✅ **免费额度**：每月 3000 封免费（100 封/天），初期够用
- ✅ **送达率高**

**备选**：Postmark（送达率最优）/ AWS SES（最便宜但配置复杂）。量大了可迁移，模板层不变。

**成本**：免费起步；超过 3000 封/月后 $20/月起。

**需要你做**：注册 Resend 账号（免费），把 API 密钥给我。

---

### 7. 部署：Vercel

**是什么**：网站存放和运行的地方。你访问 naracharm.com 时，其实是访问 Vercel 服务器上的程序。

**为什么选 Vercel**：
- ✅ **Next.js 原生平台**：一键部署、自动 HTTPS、全球 CDN、零配置
- ✅ **Preview Deployment**：每次我改代码，自动生成一个预览网址给你看效果
- ✅ **自动扩缩容**：流量来了自动扩，不用我手动管服务器
- ✅ **我全程维护**：你不懂代码没关系，部署/监控/告警我处理

**关键纪律**：
- 设带宽/请求阈值告警，超 Pro 套餐前评估迁移
- 备选：Fly.io / AWS（流量失控时迁移，Next.js 16 Adapter API 让迁移平滑）

**成本**：免费起步（Hobby 套餐）；正式商用 Pro **$20/月**（含 1TB 带宽 + 10M 请求）。

**需要你做**：注册 Vercel 账号（用 GitHub 登录），把项目连给我。或者我帮你建。

## 四、数据安全

- ✅ 所有数据存 Neon（云端，自动备份）
- ✅ 用户密码用 Better Auth 加盐哈希，绝不存明文
- ✅ 支付信息由 Stripe 处理，我们**不存任何卡号**（PCI 合规）
- ✅ GDPR：预留数据导出/删除接口（欧洲用户法定权利）
- ✅ 环境变量分 dev/prod，密钥不进代码仓库
